## 概述

```ts
import { noop, isNil, isHttpUrl, isWindow, isMacOs, isWindowsOs, isPC } from '@skyroc/utils';
```

`utils` 模块提供一组轻量的基础工具函数，覆盖三类场景：

- **空操作** — `noop`（回调占位）
- **类型守卫** — `isNil`、`isWindow`（编译期类型收窄）
- **环境检测** — `isHttpUrl`、`isMacOs`、`isWindowsOs`、`isPC`（运行时判断）

---

## noop

### 签名

```ts
function noop(): void
```

### 说明

空函数，不执行任何操作，不返回任何值。

### 用途

**作为可选回调的默认值：**

```ts
interface ButtonProps {
  onClick?: () => void;
}

const Button = (props: ButtonProps) => {
  const { onClick = noop } = props;
  // onClick 始终是函数，不需要 if (onClick) 判断
  return <button onClick={onClick}>Click</button>;
};
```

**作为初始状态的占位：**

```ts
const [handler, setHandler] = useState<() => void>(() => noop);

// 需要时替换为真实 handler
setHandler(() => realHandler);
```

**满足接口要求：**

```ts
interface EventEmitter {
  onError: (err: Error) => void;
}

const emitter: EventEmitter = {
  onError: noop, // 当前不需要处理错误，但接口要求必须传
};
```

**清理副作用占位：**

```ts
useEffect(() => {
  if (!condition) return noop; // 统一返回函数类型，避免 ts 类型不一致
  const sub = subscribe(handler);
  return () => sub.unsubscribe();
}, [condition]);
```

## isNil

### 签名

```ts
function isNil(val: unknown): val is null | undefined
```

### 说明

类型守卫函数，判断值是否为 `null` 或 `undefined`（即"nil"）。

与 `== null` 的等价关系：

```ts
isNil(val) === (val == null) // 始终为 true
```

区别在于 `isNil` 是类型守卫，在 TypeScript 中会进行类型收窄。

### 示例

```ts
import { isNil } from '@skyroc/utils';

// 基础判断
isNil(null);      // true
isNil(undefined); // true
isNil(0);         // false
isNil('');        // false
isNil(false);     // false
isNil([]);        // false
isNil({});        // false

// 作为类型守卫使用
function processValue(val: string | null | undefined) {
  if (isNil(val)) {
    // 此处 val 的类型被收窄为 null | undefined
    return '';
  }
  // 此处 val 的类型被收窄为 string
  return val.toUpperCase();
}
```

### 与其他判断方式对比

```ts
// isNil：null 和 undefined 都为 true，有类型守卫
isNil(null);         // true
isNil(undefined);    // true

// 双等号：与 isNil 等价，但无类型守卫
null == null;        // true
undefined == null;   // true

// 三等号：只判断 null
null === null;       // true
undefined === null;  // false ❌

// Boolean 转换：null / undefined / 0 / '' / false 都为 false
Boolean(null);       // false
Boolean(0);          // false（过于宽泛）

// 可选链：isNil 的一个替代用法
val?.toUpperCase() ?? '';  // 与 isNil 配合使用效果相同
```

### 常用场景

**数组过滤 nil 值：**

```ts
import { isNil } from '@skyroc/utils';

const values = [1, null, 2, undefined, 3];
const cleaned = values.filter((v): v is number => !isNil(v));
// [1, 2, 3]，类型为 number[]
```

**条件渲染：**

```tsx
const value = props.data ?? null;

return isNil(value) ? <Empty /> : <DataView data={value} />;
```

## isHttpUrl

### 签名

```ts
function isHttpUrl(url?: string): boolean
```

### 说明

判断字符串是否以 `http://` 或 `https://` 开头（大小写不敏感）。常用于区分绝对 URL 与相对路径。

### 示例

```ts
import { isHttpUrl } from '@skyroc/utils';

isHttpUrl('https://example.com');        // true
isHttpUrl('http://example.com/path');    // true
isHttpUrl('HTTP://EXAMPLE.COM');         // true（不区分大小写）
isHttpUrl('//example.com');              // false（协议相对 URL）
isHttpUrl('/api/user');                  // false（相对路径）
isHttpUrl('ftp://example.com');          // false（非 http/https）
isHttpUrl('');                           // false
isHttpUrl(undefined);                    // false
```

### 常用场景

**动态链接处理：**

```ts
// 图标或图片链接：可能是绝对 URL 或内部路径
function resolveUrl(url: string): string {
  if (isHttpUrl(url)) {
    return url; // 外部链接，直接使用
  }
  return `${import.meta.env.VITE_BASE_URL}${url}`; // 内部路径，补全 base
}
```

**菜单路由判断：**

```ts
// 菜单项 path 可能是外部链接或内部路由
function handleMenuClick(path: string) {
  if (isHttpUrl(path)) {
    openWindow(path); // 外部链接，新 tab 打开
  } else {
    router.push(path); // 内部路由，正常导航
  }
}
```

**表单 URL 字段校验：**

```ts
// 结合 REG_URL 做更严格的校验
function validateUrl(value: string): string | null {
  if (!isHttpUrl(value)) return '请输入以 http:// 或 https:// 开头的地址';
  if (!REG_URL.test(value)) return 'URL 格式不正确';
  return null;
}
```

## isWindow

### 签名

```ts
function isWindow(value: any): value is Window
```

### 说明

类型守卫，判断值是否为浏览器的 `window` 对象。内部通过 `value === value.window` 进行判断（`window.window === window` 是浏览器规范保证的）。

同时检查 `typeof window !== 'undefined'`，在 SSR / Node.js 环境中安全调用不会报错。

```ts
import { isWindow } from '@skyroc/utils';

isWindow(window);        // true（浏览器环境）
isWindow(globalThis);    // false（非 window 对象）
isWindow(null);          // false
isWindow({});            // false
```

常用于需要区分"是否传入的是 window 对象"的场景，例如封装支持 `Element | Window` 的事件监听工具。

## isMacOs

### 签名

```ts
function isMacOs(): boolean
```

### 说明

通过 `navigator.userAgent` 检测当前是否为 macOS 环境（匹配 `macintosh` 或 `mac os x`，不区分大小写）。

> 仅在浏览器环境有效（依赖 `navigator`），在 Node.js / SSR 中会抛出 ReferenceError。

```ts
import { isMacOs } from '@skyroc/utils';

isMacOs(); // true（macOS Chrome/Safari/Firefox）
isMacOs(); // false（Windows / Linux / iOS）
```

## isWindowsOs

### 签名

```ts
function isWindowsOs(): boolean
```

### 说明

通过 `navigator.userAgent` 检测当前是否为 Windows 环境（匹配 `windows` 或 `win32`，不区分大小写）。

```ts
import { isWindowsOs } from '@skyroc/utils';

isWindowsOs(); // true（Windows 系统的浏览器）
isWindowsOs(); // false（macOS / Linux / iOS / Android）
```

---

## isPC

### 签名

```ts
function isPC(): boolean
```

### 说明

通过 `navigator.userAgent` 检测当前是否为 PC 端（即非移动设备）。

检测逻辑：如果 userAgent 中**不包含**以下移动端标识，则认为是 PC：
`Android`、`iPhone`、`webOS`、`BlackBerry`、`SymbianOS`、`Windows Phone`、`iPad`、`iPod`

```ts
import { isPC } from '@skyroc/utils';

isPC(); // true（桌面浏览器）
isPC(); // false（手机 / 平板访问）
```

## 操作系统检测组合场景

`isMacOs`、`isWindowsOs`、`isPC` 最常见的使用场景是**根据操作系统展示对应的快捷键提示**：

```tsx
import { isMacOs } from '@skyroc/utils';

const ShortcutHint = () => {
  const mac = isMacOs();

  return (
    <span>
      {mac ? '⌘' : 'Ctrl'} + K 搜索
    </span>
  );
};
```

```tsx
// 快捷键绑定时区分系统
import { isMacOs } from '@skyroc/utils';

useEffect(() => {
  const handler = (e: KeyboardEvent) => {
    const modKey = isMacOs() ? e.metaKey : e.ctrlKey;
    if (modKey && e.key === 'k') {
      openSearch();
    }
  };
  window.addEventListener('keydown', handler);
  return () => window.removeEventListener('keydown', handler);
}, []);
```

```tsx
// 根据设备类型决定布局
import { isPC } from '@skyroc/utils';

const Layout = () => {
  return isPC() ? <DesktopLayout /> : <MobileLayout />;
};
```

## 函数汇总

| 函数 | 返回类型 | 是否类型守卫 | 依赖 DOM / navigator |
| --- | --- | --- | --- |
| `noop` | `void` | — | 否 |
| `isNil` | `boolean` | ✅ `val is null \| undefined` | 否 |
| `isHttpUrl` | `boolean` | 否 | 否 |
| `isWindow` | `boolean` | ✅ `value is Window` | 否（但结果无意义） |
| `isMacOs` | `boolean` | 否 | ✅ `navigator` |
| `isWindowsOs` | `boolean` | 否 | ✅ `navigator` |
| `isPC` | `boolean` | 否 | ✅ `navigator` |
