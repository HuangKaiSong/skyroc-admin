## 依赖关系图

```
                    @skyroc/ui-types
                   (ThemeColor, ThemeSize...)
                    ┌───────┴───────┐
                    ▼               ▼
             @skyroc/ui        @skyroc/native-ui
           (Web: Radix+tv)    (RN: NativeWind+tv)
                                    │
                    ┌───────────────┼───────────────┐
                    ▼               ▼               ▼
           @skyroc/native-theme  @skyroc/utils  @skyroc/color
           (NativeWind preset)    (cn 工具)     (色阶算法)
```

核心设计原则：**共享类型，独立实现**。Web 和 RN 组件共享同一套语义类型（ThemeColor、ThemeSize），但样式实现各自独立。

## 各包/应用详解

### 1. `@skyroc/ui-types` — 共享 UI 类型

**路径**: `packages/shared/ui-types/`

**为什么要提取**：`ThemeColor`、`ThemeSize` 等类型原先在 `@skyroc/ui` 的 `shared.ts` 中，属于平台无关类型。提取后 native-ui 可以直接依赖，无需引入整个 Web UI 包。

**包含类型**：

| 类型 | 说明 |
|------|------|
| `ThemeColor` | `'primary' \| 'secondary' \| 'destructive' \| 'success' \| 'warning' \| 'info' \| 'accent' \| 'carbon'` |
| `ThemeSize` | `'xs' \| 'sm' \| 'md' \| 'lg' \| 'xl' \| '2xl'` |
| `ThemeOrientation` | `'horizontal' \| 'vertical'` |
| `ThemeAlign` | `'start' \| 'center' \| 'end'` |
| `ThemeSide` | `'top' \| 'right' \| 'bottom' \| 'left'` |
| `WithClassName` | `{ className?: ClassValue }` |
| `MaybeArray<T>` | `T \| T[]` |
| `AcceptableValue` | 表单值联合类型 |
| `ClassValue` | 来自 clsx 的 re-export |

**对 `@skyroc/ui` 的影响**：原 `shared.ts` 改为从 `@skyroc/ui-types` re-export 共享类型，保留 Web 专用类型（`HTMLTag`、`PrimitiveProps`、`StyledComponentProps`、`SlotProps`、`HTMLComponentProps`）。所有现有导入路径不变，零破坏性。

### 2. `@skyroc/native-theme` — 设计令牌 + NativeWind 预设

**路径**: `packages/native/theme/`

**职责**：定义 RN 组件库的设计系统底层——颜色、间距、字体，并封装为 NativeWind v4 可用的 Tailwind 预设。

**目录结构**：

```
theme/
├── src/
│   ├── tokens/
│   │   ├── colors.ts       # 语义色（CSS 变量模式，与 ThemeColor 对齐）
│   │   ├── spacing.ts      # 4px 基准间距系统 + border-radius
│   │   └── typography.ts   # 字号、行高、字重
│   ├── presets/
│   │   └── nativewind-preset.ts   # Tailwind 预设，注入所有 tokens
│   └── types.ts            # ColorScheme, NativeThemeConfig 等
└── tailwind.config.ts      # 可直接 extend 的独立配置
```

**颜色方案**：

采用 CSS 变量 + HSL 模式，与 Web 端主题系统一致：

```css
--color-primary: 239 84% 67%;          /* 主色 */
--color-primary-foreground: 0 0% 100%; /* 主色上的文字 */
```

语义色名称与 `ThemeColor` 完全对齐：primary、secondary、destructive、success、warning、info、accent、carbon。每个色都有 `DEFAULT` 和 `foreground` 两个值。

**使用方式**：

```ts
// 消费者的 tailwind.config.ts
import { nativewindPreset } from '@skyroc/native-theme/preset';
export default { presets: [nativewindPreset], content: [...] };
```

### 3. `@skyroc/native-ui` — 核心组件库

**路径**: `packages/native/ui/`

**技术栈**：NativeWind v4 + tailwind-variants (tv) + React Native

**目录结构**：

```
ui/
├── src/
│   ├── components/
│   │   ├── button/        # Button 组件
│   │   │   ├── Button.tsx
│   │   │   ├── button-variants.ts   # tv() 定义 slots + variants
│   │   │   ├── types.ts
│   │   │   └── index.ts
│   │   ├── text/          # Typography 组件
│   │   ├── input/         # Input 组件
│   │   └── box/           # Box 布局容器
│   ├── types/
│   │   └── shared.ts      # StyledNativeComponentProps, SlotClassNames<T>
│   ├── utils/
│   │   └── cn.ts          # 复用 @skyroc/utils 的 cn (clsx + tailwind-merge)
│   └── hooks/
├── __tests__/
│   └── components/
│       └── button.test.tsx
├── jest.config.ts          # react-native preset
└── jest.setup.ts
```

**组件编写规范**（严格遵循项目 CLAUDE.md）：

- 箭头函数组件 + PascalCase
- 显式 Props interface + JSDoc 注释
- Props 在函数体内第一行解构
- 内部辅助函数用 function 声明
- 禁止 `useCallback`，限制 `useMemo`

**组件 Variant API 模式**：

每个组件用 `tailwind-variants` 的 `tv()` 定义 slots 和 variants，输出 class 字符串交给 NativeWind 编译。与 Web 端 `@skyroc/ui` 共享同一套 tv API 模式。

```ts
// 示例：button-variants.ts
const buttonVariants = tv({
  slots: {
    root: 'flex-row items-center justify-center rounded-lg',
    label: 'font-medium'
  },
  variants: {
    color: { primary: {...}, destructive: {...} },
    size:  { sm: {...}, md: {...}, lg: {...} },
    variant: { solid: {}, outline: {...}, ghost: {...} }
  }
});
```

**P0 首批组件**：

| 组件 | 说明 | 特性 |
|------|------|------|
| **Button** | 可按压按钮 | 7 色、5 尺寸、3 变体（solid/outline/ghost）、loading/disabled 状态、slot classNames |
| **Typography** | 文本排版 | 8 尺寸（xs→4xl）、4 字重、9 色 |
| **Input** | 文本输入 | 3 变体（outline/filled/underline）、3 尺寸、error 状态、leading/trailing slots |
| **Box** | 布局容器 | View 封装，接受 className |

**SlotClassNames 模式**：

Button 和 Input 使用 slot 化设计，消费者可以覆盖任意 slot 的样式：

```tsx
<Button classNames={{ root: 'rounded-full', label: 'uppercase' }}>
  Custom
</Button>
```

**测试策略**：

使用 Jest + @testing-library/react-native（非 Vitest），因为 RNTL 针对 Jest 设计，Metro transform 与 Jest 契合。根目录 `vitest.config.ts` 已排除 `packages/native/**`。

### 4. `apps/native-playground` — Expo Playground App

**路径**: `apps/native-playground/`

**职责**：组件开发调试 + 构建为 Web 静态站供文档站 iframe 加载。

**技术栈**：Expo SDK 52 + Expo Router（文件路由）+ NativeWind v4

**路由结构**：

```
app/
├── _layout.tsx              # Root layout + NativeWind provider
├── index.tsx                # 组件目录首页（列表链接到各组件）
└── components/
    ├── _layout.tsx          # Stack navigator
    ├── button.tsx           # /components/button
    ├── text.tsx             # /components/text
    └── input.tsx            # /components/input
```

每个组件 demo 有**确定的 URL 路径**，这是文档站 iframe 预览的关键。

**Metro 配置要点**：

```js
// monorepo 支持
config.watchFolders = [monorepoRoot];
config.resolver.nodeModulesPaths = [local, root];
config.resolver.disableHierarchicalLookup = true;

// NativeWind 集成
module.exports = withNativeWind(config, { input: './global.css' });
```

**运行方式**：

| 命令 | 用途 |
|------|------|
| `expo start --web` | 浏览器预览，文档站 dev 代理目标 |
| `expo export --platform web` | 静态产物，production iframe 源 |
| `expo run:ios` / `expo run:android` | 原生平台调试 |

### 5. `apps/docs` — Fumadocs 文档站

**路径**: `apps/docs/`

**职责**：组件 API 文档 + 手机壳实时预览，类似 antd-mobile 文档站效果。

**技术栈**：Next.js 15 + Fumadocs + Tailwind CSS

**核心架构 — 手机壳预览**：

```
┌─────────────────────────────────────────────────┐
│  文档页面 (Fumadocs MDX)                         │
│                                                   │
│  ┌──────────────────┐  ┌────────────────────┐   │
│  │  Props 表格       │  │  ╭──────────────╮  │   │
│  │  代码示例         │  │  │   (notch)    │  │   │
│  │  使用说明         │  │  ├──────────────┤  │   │
│  │  ...             │  │  │              │  │   │
│  │                  │  │  │   <iframe>   │  │   │
│  │                  │  │  │   Expo Web   │  │   │
│  │                  │  │  │   Preview    │  │   │
│  │                  │  │  │              │  │   │
│  │                  │  │  ├──────────────┤  │   │
│  │                  │  │  │  (home bar)  │  │   │
│  └──────────────────┘  │  ╰──────────────╯  │   │
│                         │   sticky 定位       │   │
│                         └────────────────────┘   │
└─────────────────────────────────────────────────┘
```

**PhoneFrame 组件** (`components/phone-frame/`)：
- 纯 CSS 手机壳造型（圆角边框、notch、home bar）
- 支持 iOS / Android 两种壳切换
- sticky 定位，滚动文档时保持可见
- 内嵌 `<iframe>` 指向 Expo Web 构建的组件路由

**ComponentPreview MDX 组件** (`components/mdx/`)：
- 在 MDX 中使用一行代码即可嵌入预览：

```mdx
<ComponentPreview component="button" />
```

自动拼接 iframe URL → `/native-preview/components/button`

**Dev / Prod 联调**：

| 环境 | 机制 |
|------|------|
| **开发** | Next.js rewrites 代理 `/native-preview/*` → Expo Web dev server (`localhost:8081`) |
| **生产** | 环境变量 `NEXT_PUBLIC_NATIVE_PREVIEW_URL` 指向 Expo 静态产物部署地址 |

**内容结构**：

```
content/docs/native/
├── index.mdx                    # Getting Started
├── meta.json                    # 导航树
└── components/
    ├── meta.json
    ├── button.mdx               # Button 文档 + 预览
    ├── text.mdx                 # Text 文档 + 预览
    └── input.mdx                # Input 文档 + 预览
```
