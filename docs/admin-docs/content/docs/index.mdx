Skyroc Admin React 是一个面向中后台产品的 React 应用模板。这份文档从 `apps/admin` 的实际实现出发，说明应用如何启动、路由如何生成、布局和菜单如何组合、请求与权限如何接入，以及共享包在应用里承担什么职责。

## 技术栈

| 范围 | 当前使用 |
| --- | --- |
| 应用框架 | React、Vite、TypeScript |
| 路由 | TanStack Router、约定式路由、`routeTree.gen.ts` |
| 数据与状态 | React Query、Jotai |
| UI 与样式 | Ant Design、UnoCSS、Skyroc Web UI |
| 应用基础能力 | `@skyroc/web-admin-layouts`、`@skyroc/web-admin-theme`、`@skyroc/web-admin-i18n`、`@skyroc/web-admin-notification`、`@skyroc/web-admin-vite` |
| 工具与服务 | `@skyroc/axios`、`@skyroc/utils`、`@skyroc/core-state`、`@skyroc/scripts` |

依赖入口以 `apps/admin/package.json` 为准。文档中出现的包名、脚本和能力说明都应能回到这个文件或 `packages/web/*`、`packages/@core/*` 中查证。

## 架构概览

| 层级 | 关键位置 | 职责 |
| --- | --- | --- |
| 浏览器入口 | `apps/admin/src/main.tsx` | 开发环境加载 Jotai devtools，再进入应用 bootstrap。 |
| 启动编排 | `apps/admin/src/bootstrap.tsx` | 初始化主题、后台布局、应用插件和 i18n，最后挂载 React 根节点。 |
| Provider 组合 | `apps/admin/src/App.tsx` | 组合 React Query、Jotai、开发工具、Ant Design、通知、动画、路由和全局副作用。 |
| 全局配置 | `apps/admin/src/config.ts` | 收口默认首页、路由模式、菜单图标、语言、主题和 devtools 配置。 |
| 路由与权限 | `apps/admin/src/features/router`、`apps/admin/src/features/auth` | 创建 TanStack Router，注入登录态、用户信息、首页和鉴权上下文。 |
| 菜单与布局 | `apps/admin/src/features/menus`、`apps/admin/src/pages/(admin)/layout.tsx` | 将路由、菜单分类、菜单节点转换和布局插槽接入后台应用壳。 |
| 请求服务 | `apps/admin/src/service` | 按 `urls`、`api`、`hooks`、`keys`、`types` 分层组织接口和 React Query hooks。 |
| 业务页面 | `apps/admin/src/pages` | 通过约定式目录承载登录页、后台页面、错误页、布局页和具体业务页面。 |

启动顺序可以简化为：

```text
main.tsx
  -> bootstrap.tsx
    -> setupTheme
    -> setupAdminLayouts
    -> setupAdminPlugins
    -> setupI18n
    -> render App
      -> Provider stack
      -> RouterProvider
      -> GlobalEffect
```

这个顺序很重要：主题和布局运行时能力先初始化，应用 Provider 再读取这些状态；路由 Provider 通过 `useAuth` 把登录态、用户信息和权限上下文交给 TanStack Router。
