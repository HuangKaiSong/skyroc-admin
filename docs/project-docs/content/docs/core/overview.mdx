## 定位

`packages/@core/` 是项目的**基础设施层**，存放框架无关或轻度依赖 React 的核心能力包，以及独立发布的 CLI 工具。

放在这里的标准：

- **跨平台**：Web / React Native / Node 均可使用，或仅需 React 而不依赖浏览器 API；
- **与业务无关**：属于通用基础能力；
- **职责边界清晰**：不与其他 @core 包循环依赖。

不放这里的：业务组件、页面级逻辑、特定平台 hooks（这些放 `packages/hooks`、`packages/web/*`）。

## 依赖方向

```text
types（零依赖，类型声明）
  ↑
utils（零 @core 依赖）     logger / scheduler / state / scripts（彼此独立）
  ↑
color / axios（依赖 utils）
  ↑
service（依赖 axios）
```

> 箭头表示「被依赖」方向。**禁止同层或反向依赖。** `state` 通过 `peerDependencies` 依赖 `jotai >= 2.0.0` 与 `react >= 18.0.0`，不在内部依赖图里。
