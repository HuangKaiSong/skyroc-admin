`packages/@core` 是项目的基础设施层，用来承载跨平台、低业务耦合、可以独立发布或复用的核心能力包。

这些文档面向在业务应用、组件库、文档站和工程工具中复用基础能力的开发者，重点说明每个包解决什么问题、应该从哪里导入、以及与其他核心包的关系。

## 文档入口

<Cards>
  <Card title="@skyroc/utils" href="/docs/utils/overview">
    平台无关工具函数集合，包含日期、存储、加密、事件、数组、对象、并发去重和浏览器专用工具。
  </Card>
  <Card title="@skyroc/axios" href="/docs/axios">
    类型安全的 Axios 请求客户端工厂，提供业务响应适配、错误处理、Token 刷新和请求取消能力。
  </Card>
  <Card title="@skyroc/service" href="/docs/service">
    请求与查询基础设施，通过平台适配器接入 UI、鉴权、导航和 React Query。
  </Card>
  <Card title="@skyroc/core-state" href="/docs/state">
    基于 Jotai 的状态管理封装，支持全局 store、持久化原子和组件外访问。
  </Card>
  <Card title="@skyroc/scheduler" href="/docs/scheduler">
    协作式任务调度中枢，用统一心跳管理初始化任务、周期任务和监听器。
  </Card>
  <Card title="@skyroc/color" href="/docs/color">
    面向设计系统和主题引擎的色彩工具包，提供调色板生成、颜色转换和无障碍对比度能力。
  </Card>
  <Card title="@skyroc/scripts" href="/docs/scripts">
    Skyroc Admin CLI 工具集，覆盖提交规范、发版、changelog、依赖更新和构建产物清理。
  </Card>
</Cards>

## 如何选择

| 场景 | 优先使用 |
| --- | --- |
| 通用工具函数、日期、存储、事件、下载等能力 | `@skyroc/utils` |
| 只需要一个类型安全的 HTTP 客户端 | `@skyroc/axios` |
| 需要业务错误处理、Token 刷新、React Query 和平台适配 | `@skyroc/service` |
| 需要跨组件或组件外访问的状态管理 | `@skyroc/core-state` |
| 需要统一调度启动任务、定时任务或监听器 | `@skyroc/scheduler` |
| 需要生成主题色、暗色模式色板或做 WCAG 对比度检测 | `@skyroc/color` |
| 需要项目自动化命令、提交引导、发版或 changelog | `@skyroc/scripts` |

## 依赖关系

```text
types
  ↑
utils
  ↑
color / axios
          ↑
        service

state / scheduler / scripts 相对独立，按具体业务场景引入。
```

依赖方向应保持从基础能力到上层组合能力，不应让底层包依赖业务包、UI 包或上层服务包。
